Let's start with some basic organization of a project. We will call it "Stack Containers" - seems to be clear and simple. At first, let's prepare some directories. For directories and non-source files I like to use kebab-case notation. It's kinda easy to read.

$ mkdir stack-containers
$ cd stack-containers

For sure we will have some source files and some tests. I think the implementation will be header only (since the containers are going to use templates) so there is no need for a separate include folder (which I would normally add).

$ mkdir source
$ mkdir test

Since we are using CMake, let's add a configuration file.

$ touch CMakeLists.txt
$ code CMakeLists.txt

First, let's configure CMake, project name, and C++ specific information. We specify that we want to use standard C++17 and we pass some additional flags for the compiler: turn on all standard warnings (-Wall), turn on additional warnings (-Wextra), turn every warning into an error (-Werror), and also... turn on more warnings! (-pedantic)

cmake_minimum_required(VERSION 3.16)
project("Stack Containers" CXX)

set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_FLAGS "${CMAKE_CXX_FLAGS} -Wall -Wextra -Werror -pedantic")

We could probably use older version of CMake but that's the one that I've got so let's keep it. I also really don't understand why do they use variable names with CXX instead of CPP. Oh well... The header files will be kept in source folder, so let's notify CMake about where to look for them.

include_directories(source)

As I promised - I will follow TDD rules. Let's start with tests. CMake has a nice feature called CTest which is helpful to distinguish compiled tests from other binaries. CMake also comes with a function add_test which is not so useful on its own but with a couple of helper functions it will do the job. Let's see.

function(target_output_directory binary dir)
  set_target_properties(${binary} PROPERTIES RUNTIME_OUTPUT_DIRECTORY ${dir})
endfunction()

function(add_new_test target)
  add_executable(${target} ${ARGN})
  target_output_directory(${target} tests)
  add_test(${target} tests/${target})
endfunction()

The CMake function declaration is somewhat awkward. First you use function keyword, then you pass its name as argument of function function, then you pass recognizable parameters of that function. In our case we declared two helper functions: target_output_directory - which job is to change the path where the binary target will go, and add_new_test - which is more complicated. First, it adds executable target that will be build from any source files passed as additional arguments to function add_new_test. Then, with a help of previous helper function, it changes the default output path to test directory (it is not directory with test source files but rather out-of-build test directory for binaries). In the end the function marks newly created test binary as a testable entity. Huh!

Now let's add our first test.

include(CTest)
enable_testing()
if(BUILD_TESTING)
  add_new_test(array-test test/ArrayTest.cpp)
endif()

We need to include CTest in order to easily turn on/off compilation of test and testing itself. BUILD_TESTING is set to true by default but we can still disable it by passing -DBUILD_TESTING=false argument to CMake. Let's add a dummy source file for our test.

$ echo "int main() { return 0; }" > test/ArrayTest.cpp

Now let's try to build it. Remember to always do out of source builds. You don't want to clutter your beautiful code and project layout with some compile-time garbage!

$ mkdir build
$ cd build
$ cmake ..
